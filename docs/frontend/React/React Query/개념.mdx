## 1. 기본 개념

참고할만한 깃허브 레포지토리 : https://github.com/ssi02014/react-query-tutorial

React Query는 서버 상태 관리를 위한 라이브러리<br/>
React Query는 서버 상태(Server State)와 클라이언트 상태(Client State)를 구분

### 서버 상태
서버에 저장되고 API를 통해 가져오는 데이터 (예: 할 일 목록)
서버에 저장되지만 클라이언트에 데이터가 있어야만 표시할 수 있는 데이터 (게시물 등)

### 클라이언트 상태
웹 브라우저 세션에 종속되는 모든 정보
다크모드 설정, 언어 설정 등
클라이언트에서만 관리되는 UI 상태 (예: 필터링 옵션)

### 리액트 쿼리 캐시
서버 데이터가 필요할때 React-Query-Cache 에 요청
여기에 없으면 그제서야 서버에서 받아옴

### 리액트 쿼리 캐싱 방법
- 명령형 : 서버에서 싹 새로 가져옴
- 선언형 : 리프레시 되는 조건을 선언해줌. 조건을 만족하면 서버에서 데이터를 가져옴

### 용어 및 특징
- 프리페칭 : 사용자가 가져올 것으로 예상되는 데이터를 미리 가져옴
- 중복 제거 : 리액트 쿼리는 여러 요청이 들어와도 중복을 제거 해줌


## 2. 사용법
1. 쿼리 클라이언트를 생성 (쿼리와 캐시를 관리)
2. 쿼리 클라이언트 프로바이더 생성 (쿼리 클라이언트 등 여러가지 한꺼번에 제공)
3. useQuery 등 훅 사용
